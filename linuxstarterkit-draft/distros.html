Picking Your First Linux Distro

Picking your first Linux distribution can be a daunting task. You might face some questions such as but not limited to:

  • What is a linux distribution?
  • Why are there so many distros?
  • Does it matter which one I use?
  • Which Linux distribution is better?
  • Why are people telling me to “Install Gentoo”?

These are completely normal questions to have if you are a new user. There are a lot of moving parts in the world of Linux Distributions. To illustrate this here is DistroWatch's "Periodic Table of Distros" (Figure 1). To make matters worse the periodic table is actually a simplified and slimmed down picture of Linux distros. For a full family tree of linux distributions see DistroWatch's family tree image. It's really mind blowing to look at.

Despite this it is actually quite easy to break these down for a new user. Only a few of these distributions are truly user-friendly and general-purpose so only a few are even worth considering for a fist time user.

What is a Linux distribution?

To be clear, "distro" is just short for "distribution".

In short a distribution is just a variation of linux configured and implemented differently. The difference often being how the user interacts with the system. To demonstrate, (Figure 2) below we have Ubuntu 18.04 (left) and OpenSUSE Tumbleweed (right). You can use the slider to compare how their interfaces differ.


Ubuntu and OpenSUSE are examples of two different distros developed by two different companies, Canonical Ltd. and SUSE respectively. They take very different approaches to the user interface. While both interfaces include most of the same features functionally, OpenSUSE uses the KDE interface which follows a similar format of the Windows UI while Ubuntu uses the GNOME interface which has it's own philosophy and design for graphical user interfaces.

The types of graphical interfaces for Linux Operating Systems such as GNOME and KDE mentioned earlier, are referred to as Desktop Environments or DEs for short. The Desktop Environment is usually developed by a seperate organization but then reconfigured and shipped by the distro. You will notice the KDE foundation and the GNOME project are seperate entities from Canonical Ltd. and SUSE. This raises and important question...

Why are there so many distros?

As we've discussed there are a lot of Linux distros and organizations involved in their development. So why is that the case? Well to understand this it helps to know about the history of Linux. There is an overview on the home page. The gist of it is that the open source nature of Linux means anyone can study and modify the code and redistribute it under certain conditions. This allows companies such as Coninical and SUSE to distribute it with commercial services. And so there are now thousands of Linux distros each with a special niche.
